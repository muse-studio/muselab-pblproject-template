\section{はじめに}
人々は古来より，思いを様々な方法で伝えてきた．その方法の中でも，文字で伝えるということに着目したい．文字で伝える方法には，手紙など書いて伝える，口に出して言葉で伝える，歌で伝えるといった方法がある．\par
本研究では，言葉で伝える，歌で伝えるという二つに着目したい．二つとも，「口に出して，言葉で伝える」ということは一致しているが，表現方法は異なる．\par
この二つの表現方法は、どちらも「音」を通じて情報や感情を伝える手段として用いられる。しかし、発話は主に会話や朗読の場で情報伝達や感情表現を目的とするのに対し、歌は旋律やリズムを伴うことで芸術的要素を持ち、より感情やメッセージを強調する手段となる。このように、共通点がありつつも異なる性質を持つ発話と歌について、より詳細に分析する必要がある．\par
本研究では，発話表現と歌の表現を，「音での表現」という観点から比較し，両者に違いがあるのかを考察していく．

\section{研究概要}

%3.1
\subsection{分析対象}\label{taisyou}
本研究で分析対象となる，楽譜から抽出する音楽の要素を以下に挙げる．\par
\begin{musequote}
  \begin{itemize}
    \item 楽曲名：どの楽曲か＋同曲内の出現回数
    \item BPM：秒数・累積秒に必要
    \item 拍子：分析に必要
    \item 調号：分析に必要
    \item 音符の種類：音符か休符か
    \item MIDI：音の高さ
    \item 拍数・累積拍：相対時間の基準
    \item オフセット：曲の開始からの拍の長さ
    \item 秒数・累積秒：絶対時間の基準
    \item 歌詞：分析に必要
  \end{itemize}
\end{musequote}
また，楽譜データとは別に楽曲のメタデータを作成した．内容は以下の通りである．\par
\begin{musequote}
  \begin{itemize}
    \item アーティスト・作詞・作編曲者：同じフレーズを複数回使用しているアーティストはいるのか
    \item 歌い手：男女，グループかどうか
    \item 発売日：発売年ごとの傾向の分析
  \end{itemize}
\end{musequote}
抽出した楽譜のデータとメタデータ，それぞれにIDを付けることによって，連動して分析を行うことができる．

%3.2
\subsection{楽譜データ}\label{sec:gakuhu}
音楽的要素を抽出するため，まず楽曲の楽譜データの作成を行う．手順を以下に示す．楽譜データの形式にはMusicXML\cite{musicxml}を使用した．

%3.2.1
\subsubsection{歌詞の選定}
まず，研究対象になる歌詞の選定から行う．選定は，\ref{akanesan}で記述されている基準で行う．以下にその基準を記述する．\par
\begin{enumerate}
  \begingroup
  \renewcommand{\labelenumi}{\arabic{enumi})}
  %\renewcommand{\labelenumii}{\alph{enumii}).}
  \item ひとつの単語だけでなく，名詞や助詞，動詞が含まれること．（複数の単語を組み合わせたフレーズを用いることで，拍子の変化や音の高さの変化に特徴が現れやすいと思われるためである）
  \item フレーズから想起される文脈がある程度一般的であること．
  \item 日本の楽曲で少なくとも100 曲は歌われていること．
        \endgroup
\end{enumerate}
本研究では，これらに合致した楽曲を分析対象とする．

%3.2.2
\subsubsection{対象楽曲}\label{3.2.2}
\ref{akanesan}では，この基準に一致した詩句，「あなたにあいたくて」を含む楽曲が 157 曲見つかっていた．\par
また今回新たに，上記の基準を参考に詩句の選定を行った．結果，「ひかりのほう」という詩句が基準に一致した．表記ゆれの内訳を以下に示す．\par
\begin{itemize}
  \item 「光の方」…151曲
  \item 「光のほう」…25曲
  \item 「ひかりの方」…4曲
  \item 「ひかり（ヒカリ）のほう」…5曲
\end{itemize}
上記の通り，「ひかりのほう」の詩句が含まれる楽曲は，2024年10月15日現在で合計 185 曲ある．以上の歌詞の選定により，「あなたにあいたくて」の含まれる楽曲157 曲，「ひかりのほう」の含まれる楽曲185曲を，以下の作成方法で収集する．

\subsubsection{収集方法}
本研究では音楽表現を分析するため，各楽曲の旋律情報が必要である．そのため，\ref{3.2.2}で示した分析対象の各楽曲のフレーズが書かれた楽譜を入手する必要がある．\par
入手方法として以下が挙げられる．
\begin{musequote}
  \noindent
  \textbf{出版されている楽譜を入手する}\par
  入手した後，楽譜制作ソフトを用い，当該フレーズ箇所の電子楽譜データを作成する．\par
  \noindent
  \textbf{音源のみ入手できた楽曲}\par
  音源を聴きながら 楽譜制作ソフトを用い，当該フレーズ箇所の電子楽譜データを作成する．\par
  \noindent
  \textbf{どちらもできなかった楽曲}\par
  楽譜データを作成することが不可能なため，本研究では対象外とする．\par
\end{musequote}

以上の手順を用いて楽譜データを作成する．\par
また，本研究で使用する各フレーズの入手について，以下に記す．

\begin{musequote}
  \noindent
  \textbf{「あなたにあいたくて」フレーズについて}\par
  本研究では，先行研究で使用されていた「あなたにあいたくて」フレーズ群と，追加で入手できた4曲7フレーズを合わせ，合計132曲分176フレーズを本研究の分析対象とする．\par
  \noindent
  \textbf{「ひかりのほう」フレーズについて}\par
  全185曲中，18曲は音源が見つけられず，楽譜を作成することができなかった．また，音源は見つかったが楽譜に起こすのが難しかったものが4曲，作成途中が32曲ある．現在作成できている楽譜は131曲である．\par
\end{musequote}

%3.4.1
\subsubsection{楽譜データセット}
MusicXMLをPython上で読み込む際に，music21\cite{music21}というライブラリを用いて変換を行った．\par
その際，音符の長さ（以下，音長とする）はmusic21の“quarterlength”を用い，四分音符の長さをを1として変換した．例えば，“quarterlength”を用いると，二分音符は2，符点八分音符は0.75となる．また，music21にはオフセットという指標もあり，これは楽譜の開始からの四分音符の個数をカウントするものである．\par
これらを\figref{gakuhu}の楽譜を例に行うと，表\ref{quart}のようになる．\par
また，BPMを首都良くする際に，“八分音符=180”などの，四分音符基準でないBPMを取得してしまう．そのため，
\begin{displaymath}
  \text{取得したBPM} \times \text{BPMの基準の音符\\(＝の左側を四分音符に変換した数字)}\\
  \text{↑これを改行してみせたい}
\end{displaymath}
で算出した．“八分音符=180”だと，八分音符は四分音符を1とすると0.5となるため，\[180 \times 0.5 = 90\]となる．
